调试技术的应用

调试技术一般有以下两个用途：

  1. 处理错误
    • 快速定位错误位置
    • 找到无法目测出来的错误原因
  2. 理解程序
    • 跟踪程序运行轨迹
    • 观察数据变化

环境

Python PDB模块

  • 支持设置中断
  • 支持源码级单步调试
  • 查看堆栈内容
  • 查看源码
  • 计算任意当前堆栈中的代码
  • 纯Python实现

PDB(1)启动并设置断点

  1. 在待调试的程序中添加断点 
    import pdb
pdb.set_trace()
  2. 执行带调试的程序 
    python demo.py

PDB(2)断点设置技巧

可以优先考虑在以下位置设置断点

  • 程序第一条指令前(建议)
  • 代码思路混乱处
  • 函数入口

通过变量控制调试是否开启 

DEBUG = true
if DEBUG==True:
  import pdb
  pdb.set_trace()

PDB(3)一般断点与临时断点

  1. 设置一般断点 
    (pdb) break 13
(pdb) break fib
  2. 设置临时断点
    • 临时断点的参数与一般断点相同
    • 中断一次后自动删除 
    (pdb) tbreak 13

PDB(4)条件断点

条件断点

  • 忽略一定次数 
    (pdb) ignore <bpnumber> 3
  • 条件成立时中断 
    (pdb) condition <bpnumber> [condition]

PDB(5)断点使能(Enable)

设置断点使能(Enable)

  • 开启断点 
    (pdb) enable <bpnumber> [bpnumber]
  • 禁用断点 
    (pdb) disable <bpnumber> [bpnumber]

PDB(6)程序控制

PDB支持单步执行、返回当前函数、继续执行等控制程序流程的指令 

(pdb) continue
(pdb) next
(pdb) step
(pdb) return
(pdb) jump <lineno>

PDB(7)堆栈相关

显示当前位置 

(pdb) where
(pdb) bt

向上一层栈移动 

(pdb) up

向下一层栈移动 

(pdb) up

PDB(8)显示信息

显示源代码 

(pdb) l
(pdb) ll

打印表达式 

(pdb) p n
(pdb) pp n

PDB(9)查询及交互

查看各种信息 

(pdb) b
(pdb) display
(pdb) print <expr>
(pdb) retval
(pdb) args

执行Python指令 

(pdb) !n=3

PDB(10)神奇的commands

每次中断时自动执行指令序列 

(pdb) commands [bpnumber]
(com) ...
(com) ...
(com) end
(pdb)
